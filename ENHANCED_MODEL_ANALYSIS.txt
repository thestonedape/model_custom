SECTION 1 — WHAT BELT-ENHANCED IS
================================================================================

The enhanced model is an improved version of the BELT replica with 7 additions:

  1. Label Smoothing (eps=0.1) instead of hard cross-entropy
  2. AdamW optimizer (lr=5e-4, weight_decay=0.01) instead of SGD
  3. Warmup + Cosine Annealing LR (5 epoch warmup, min_lr=1e-7)
  4. Gradient Clipping (max_norm=1.0)
  5. MixUp augmentation (alpha=0.2)
  6. Stochastic Depth / DropPath (rate=0.1) in D-Conformer blocks
  7. Multi-Sample Dropout in classifier head (5 samples)

Architecture is identical to the BELT replica:
  D-Conformer (6 blocks, d_model=840) → VQ (K=1024) → MLP Classifier (500 classes)
  Total parameters: 112,821,180

SECTION 3 — BUG LOG (8 bugs fixed before stable training)
================================================================================

BUG 1: AttributeError — DConformer attribute name wrong
  Code:    self.encoder.layers
  Fix:     self.encoder.conformer_blocks
  Why:     DConformer uses conformer_blocks not layers internally

BUG 2: YAML float parsing — scientific notation read as string
  Code:    lr: 5e-4  →  TypeError: '<=' not supported between str and float
  Fix:     lr: 0.0005  (plain decimal in enhanced_config.yaml)
  Also:    Added float() cast in optimizer setup as safety net

BUG 3: LabelSmoothingCrossEntropy wrong kwargs
  Code:    LabelSmoothingCrossEntropy(num_classes=500, smoothing=0.1)
  Fix:     LabelSmoothingCrossEntropy(epsilon=0.1)
  Why:     training/enhanced_losses.py uses 'epsilon', not 'smoothing'

BUG 4: VectorQuantizerLoss class does not exist
  Code:    criterion_vq = VectorQuantizerLoss()
  Fix:     criterion_vq = None  (VQ loss comes from model.forward() directly)

BUG 5: UnicodeEncodeError — Windows CP1252 terminal
  Issue:   Print statements contained Greek letters (ε, α, γ)
  Fix:     Replaced with ASCII equivalents (eps, alpha, gamma) via PowerShell

BUG 6: Batch is tuple not dict
  Code:    batch['eeg'], batch['label'], batch['word_embedding']
  Fix:     eeg_data, labels, words = batch
  Why:     ZuCo dataset returns (eeg_tensor, label_int, word_str) tuple

BUG 7: ContrastiveLoss signature mismatch
  Code:    criterion_cl(quantized, word_embeddings, labels)
  Fix:     criterion_cl(quantized, list(words))
  Why:     ContrastiveLoss.forward takes (eeg_features, words_list) — no labels

BUG 8: IndexError — VQ got 1D input
  Issue:   encoded.mean(dim=1) on 2D [batch,840] → [batch] (1D)
  Fix:     Conditional pooling: if encoded.dim()==3 → mean(dim=1) else pass through
  Why:     EEG input is 2D [batch,840], no sequence dimension

SECTION 4 — VQ LOSS DIVERGENCE ISSUE (critical)
================================================================================

ATTEMPT 1 (lambda_vq=1.0):
  Symptom: VQ loss grew from 0.4 → 240+ over 100 batches
  Raw MSE between random codebook (near-zero init) and encoder outputs (~4-16 mag)
  explodes as CE drives encoder outputs to larger magnitudes.
  Result:  Total loss = CE(~6) + 1.0 * VQ(240+) → gradient clipping suppressed CE

ATTEMPT 2 (lambda_vq=0.1):
  Same explosion, just 10x smaller contribution — still diverging

ATTEMPT 3 (lambda_vq=0.0):
  VQ loss disabled. BUT: model still routes through VQ layer.
  Random codebook snaps encoder outputs to random buckets every forward pass.
  Codebook has ZERO gradient (lambda_vq=0 → no VQ loss → frozen at random init).
  Straight-through estimator passes CE gradients back to encoder but encoder
  must fight fixed random discretization every step.
  Result:  Plateau at 29.7-29.9% for ALL 13 epochs (flat from epoch 2 onwards)
  Smoking gun: Val Top-1 FROZEN at exactly 6.94% across all 13 epochs

PLATEAU RUN DATA — Full 13-epoch table (VQ active but frozen, lambda_vq=0.0):

  Epoch | Train Loss | Top-10  | Notes
  ------|-----------|---------|------
    1   |  5.7681   | 29.73%  | Best saved
    2   |  5.5398   | 29.95%  | Best saved ← peak of entire run
    3   |  5.5346   | 29.84%  |
    4   |  5.5336   | 29.80%  |
    5   |  5.5264   | 29.90%  | Warmup ended (LR peaked at 5e-4)
    6   |  5.5234   | 29.74%  | Cosine decay began

   13   |  5.5158   | 29.90%  | Stopped here

  Observations:
  - Train loss delta epochs 2→13: only 0.024 (5.5398 → 5.5158) — essentially zero
  - Val Top-10 range across all 13 epochs: 29.73% – 29.95% (0.22% spread)
  - Top-1 frozen at EXACTLY 6.94% on EVERY epoch — impossible without a bug
  - Confirmed root cause: VQ codebook frozen at random init acts as fixed hash
    function, classifier learns to map random buckets → words, but encoder
    cannot meaningfully update because its outputs are always discretized to
    the same random codes. Straight-through gradient exists but encoder is
    fighting a fixed destructive bottleneck.

CURRENT FIX — VQ bypass (use_vq=False):
  When lambda_vq=0, encoder output passes DIRECTLY to classifier, VQ skipped.
  Clean gradient flow: CE loss → classifier → encoder (no discretization bottleneck)
